# Claude Code Development Guide for EasyToFin

This document provides guidance on using Claude Code as your AI development assistant for the EasyToFin project.

## 🏗️ Project Architecture

### Technology Stack Overview

**Frontend (Client)**
- React 19 with TypeScript
- Vite for development and building
- TanStack Query for data fetching
- Wouter for routing
- Radix UI + Tailwind CSS for UI

**Backend (Server)**
- Express server with tRPC
- Drizzle ORM for database
- MySQL for data persistence
- JWT for session management

**Key Design Patterns**
- **Type-Safe APIs**: tRPC ensures end-to-end type safety
- **Router Pattern**: Modular tRPC routers in `server/routers/`
- **Database-First**: Schema defined in `drizzle/schema.ts`
- **Component Composition**: Reusable React components

## 📝 Code Patterns & Best Practices

### tRPC Procedure Pattern

```typescript
// server/routers/example.ts
import { z } from "zod";
import { router, publicProcedure, protectedProcedure } from "../trpc";

export const exampleRouter = router({
  // Public endpoint
  getData: publicProcedure
    .input(z.object({ id: z.number() }))
    .query(async ({ input, ctx }) => {
      // Logic here
      return { data: "result" };
    }),

  // Protected endpoint (requires auth)
  updateData: protectedProcedure
    .input(z.object({ id: z.number(), value: z.string() }))
    .mutation(async ({ input, ctx }) => {
      // ctx.user available after authentication
      // Logic here
      return { success: true };
    }),
});
```

### React Component Pattern

```typescript
// client/src/components/Example.tsx
import { useState } from "react";
import { trpc } from "@/_core/trpc";

interface ExampleProps {
  userId: number;
}

export function Example({ userId }: ExampleProps) {
  const { data, isLoading } = trpc.example.getData.useQuery({ id: userId });
  const updateMutation = trpc.example.updateData.useMutation();

  if (isLoading) return <div>Loading...</div>;

  return (
    <div>
      {/* Component JSX */}
    </div>
  );
}
```

### Database Query Pattern

```typescript
// server/db.ts
export async function getExampleById(id: number) {
  const db = await getDb();
  if (!db) return undefined;

  const result = await db
    .select()
    .from(exampleTable)
    .where(eq(exampleTable.id, id))
    .limit(1);

  return result.length > 0 ? result[0] : undefined;
}
```

## 🔐 Authentication System

### Multi-Method Authentication Flow

The project supports three authentication methods:

1. **Phone OTP** (`server/routers/phone-auth.ts`)
   - User enters phone number
   - SMS OTP sent via Twilio
   - OTP verification creates session

2. **Email OTP** (`server/routers/email-auth.ts`)
   - User enters email
   - Email OTP sent via SendGrid
   - OTP verification creates session

3. **Gmail OAuth** (`server/routers/gmail-auth.ts`)
   - Google Sign-In button
   - OAuth callback handler
   - Session creation

### Role-Based Access Control

```typescript
// Protected procedure checks user role
export const adminProcedure = protectedProcedure.use(async ({ ctx, next }) => {
  if (!["admin", "super_admin"].includes(ctx.user.role)) {
    throw new TRPCError({ code: "FORBIDDEN" });
  }
  return next();
});
```

### Session Management

Sessions are managed with:
- HTTP-only cookies
- JWT tokens (jose library)
- Automatic timeout (30 minutes default)
- "Remember Me" option (30 days)

## 🐛 Common Gotchas

### 1. Environment Variables

Always prefix client-side env vars with `VITE_`:
```
VITE_GOOGLE_CLIENT_ID=xxx  # ✅ Accessible in client
GOOGLE_CLIENT_SECRET=xxx   # ❌ Server-only
```

### 2. tRPC Type Inference

Import types from the router, not the database:
```typescript
// ❌ Wrong
import { User } from "../drizzle/schema";

// ✅ Correct
import { trpc } from "@/_core/trpc";
type User = RouterOutput["user"]["getMe"];
```

### 3. Database Connections

Always check if DB is available:
```typescript
const db = await getDb();
if (!db) {
  throw new Error("Database not available");
}
```

### 4. Session Cookies

Ensure cookie settings match environment:
```typescript
// Development
sameSite: "lax"

// Production (HTTPS)
sameSite: "none",
secure: true
```
